Image: `apps/marketing/public/assets/linkedin/page-02-governance-engine.png`

Who said this?
When was it recorded?
Was it reviewed?
Is it still current?
What did it replace?

OpenBean is built around those questions.

Every update is kept in append-only history. Trusted knowledge is derived from that history. Review decisions are preserved instead of overwritten.

The goal is not just to find information.

The goal is to govern what becomes trusted organizational knowledge.

Learn more: https://openbean.xyz

Title:

```text
From Idea to Dogfooding: The First OpenBean Milestones
```

Subtitle:

```text
How an open-source governance layer for organizational AI memory moved from philosophy to a working product loop.
```

Article:

```markdown
OpenBean started with a simple concern:

AI tools are becoming better at helping teams produce knowledge, but they should not silently decide what an organization knows.

That distinction matters.

Chat history is not organizational memory. A retrieved note is not automatically trusted knowledge. A useful fact from an AI assistant still needs context, review, ownership, and history.

That is the foundation behind OpenBean.

AI proposes. Humans govern. Organizations remember.

## Milestone 1: The Principle

The first milestone was not code.

It was deciding what OpenBean should not be.

OpenBean is not another chatbot, RAG framework, vector database, or AI memory SDK. Those systems are mostly concerned with retrieval.

OpenBean is concerned with governance.

The core question is not only: can the system find something relevant?

The deeper question is: should this become trusted organizational knowledge?

## Milestone 2: The Governance Model

For memory to be useful inside an organization, it has to answer basic questions:

Who said this?
When was it recorded?
Was it reviewed?
Is it still current?
What did it replace?
Can we see the history?

OpenBean is built around append-only history, human review, and provenance.

Memory is not silently overwritten. Updates are preserved. The current answer is derived from the history instead of replacing it.

That design is slower to build than a simple retrieval layer, but it is the design organizational memory needs.

## Milestone 3: Verification

The next milestone was proving the governance model against real behavior.

OpenBean's acceptance suite verifies the core invariants against a live stack, including append-only history, review routing, isolation, and provenance.

This matters because governance cannot only be a sentence in a README.

If AI tools can propose memory, then the system has to prove that review cannot be bypassed and history cannot be silently rewritten.

## Milestone 4: Dogfooding

OpenBean is now being used to build OpenBean.

That changed the project.

The product is no longer only exercised through tests and design documents. It is being used during real development work.

That has already produced useful friction: review-state bugs, search behavior lessons, health-check work, backup and restore drills, and clearer setup flows.

Those are not side quests. They are the product becoming real.

## Milestone 5: First Trusted Recall

The most meaningful milestone so far was small.

During development, OpenBean recalled trusted project knowledge that replaced manually searching configuration files.

That is the first proof point the project cares about.

Not loose chat history.
Not vague retrieval.
Not an AI silently deciding what is true.

A memory was proposed during real work, reviewed through a human workflow, stored with provenance, and recalled later as trusted knowledge.

Small proof point, real value.

## What Comes Next

OpenBean is still early.

The work now is to keep tightening the loop: local setup, review, search, provenance, backup, restore, and the developer experience around connecting AI tools through MCP.

The goal is not hype.

The goal is a self-hosted, open-source memory layer that engineering teams and their AI tools can both trust.

OpenBean: https://openbean.xyz
GitHub: https://github.com/openbean/openbean
```
